# Autonomía de Trabajo

## Principios de Ejecución Autónoma

- **NO pedir confirmación** para continuar con tareas secuenciales cuando el usuario ha dado una instrucción clara
- **Trabajar de forma continua** en las tareas de la especificación hasta completar la fase actual
- **Solo preguntar** cuando hay decisiones de diseño ambiguas o errores críticos
- **Informar progreso** de forma concisa sin solicitar aprobación para cada paso

## Contexto del Proyecto Music Info Enhancement

- El usuario quiere que implemente las tareas secuencialmente sin interrupciones
- Continuar con la Fase 3: Navigation System después de completar Fase 2
- Implementar todas las subtareas de cada fase de forma autónoma
- Solo reportar cuando una fase completa esté terminada

## Estilo de Comunicación

- Ser directo y eficiente
- Mostrar progreso con acciones concretas
- Evitar preguntas innecesarias como "¿continúo?" o "¿está bien?"
- Proceder con confianza basándose en la especificación existente
